4 Kernthema

Das Harness im Detail

Hier entscheidet sich, ob ein Agent ein Demo-Spielzeug bleibt oder produktionsreif wird. Memory, Tools, Skills, MCP, Observability, Guardrails – und warum das Harness das Thema 2026/2027 ist.

Nach diesem Modul kannst du …

  • die Hauptbausteine eines Harness benennen und ihre Aufgabe erklären
  • verstehen, was MCP ist und welches Problem es löst
  • den Unterschied zwischen Tools, Skills und Memory sauber ziehen
  • begründen, warum „Harness Engineering" zur eigenen Disziplin geworden ist

4.1 Was ist das Harness?

Das Harness ist die komplette Software-Infrastruktur, die ein LLM umschließt: der Orchestrierungs-Loop, die Tools, das Gedächtnis, das Kontext-Management, die Zustands-Persistenz, die Fehlerbehandlung und die Guardrails. Der Begriff wurde Anfang 2026 zum Standardvokabular – das Konzept gab es schon länger. Kurz: Das Harness ist alles, was aus einem rohen Modell ein verlässlich arbeitendes System macht.

4.2 Die Bausteine

🧠 Memory — Gedächtnis

Das Modell ist zustandslos. Memory ist das, was Kontinuität schafft. Man unterscheidet grob: Kurzzeitgedächtnis (der Gesprächsverlauf im aktuellen Kontextfenster) und Langzeitgedächtnis (Fakten, die über Sessions hinweg überleben – z. B. in Dateien, einer DB oder einem Vektorspeicher). Die Königsdisziplin: Was kommt ins Kontextfenster und was bleibt draußen? Genau das ist Context Engineering (Modul 6 & 8).

🔧 Tools — Werkzeuge

Tools sind die Hände des Agenten. Sie werden als Schemata definiert (Name, Beschreibung, Parametertypen) und in den Kontext des LLM injiziert, damit das Modell weiß, was verfügbar ist. Die Beschreibung eines Tools ist überraschend wichtig: Ein schlecht beschriebenes Tool wird falsch oder gar nicht genutzt. Tool-Design ist halb Doku-Schreiben, halb API-Design.

📦 Skills — Fertigkeiten

Skills sind verpackte, wiederverwendbare Anleitungen oder Prozeduren – „so erstellst du einen Quartalsbericht", inklusive nötiger Schritte, Vorlagen und ggf. zugehöriger Tools. Wo ein Tool eine einzelne Funktion ist, bündelt ein Skill Wissen über ein Vorgehen. Sie lassen den Agenten komplexe, domänenspezifische Aufgaben zuverlässig wiederholen, ohne dass alles im System-Prompt steht.

🔌 MCP — Model Context Protocol

Damit Tools verlässlich und wiederverwendbar werden, definiert Anthropics Model Context Protocol (MCP) eine standardisierte, JSON-RPC-basierte Schnittstelle zwischen Modellen und externen Ressourcen. Die Analogie, die sich durchgesetzt hat: MCP ist der „USB-C-Anschluss" für KI – statt für jede Kombination aus Agent und System einen eigenen Adapter zu bauen, sprechen beide Seiten dasselbe Protokoll. Ein MCP-Server für Jira, Slack oder eine interne DB lässt sich so an beliebige MCP-fähige Agenten anstecken.

Tool vs. Skill vs. MCP – die saubere Abgrenzung

Tool = eine einzelne aufrufbare Funktion. Skill = verpacktes Vorgehenswissen, das mehrere Tools und Schritte bündelt. MCP = das Protokoll, über das Tools/Ressourcen standardisiert angebunden werden. MCP ist also kein Werkzeug, sondern die Steckdosennorm.

📊 Observability — Sicht ins Innere

Observability ist die durchgängige Sichtbarkeit jedes Schritts, den ein Agent macht: LLM-Calls, Tool-Aufrufe, Retrieval, Planungsentscheidungen. Ohne sie fliegt ein produktiver Agent blind. Weil das so zentral ist, bekommt es ein eigenes Modul (Modul 5).

🛡️ Guardrails — Leitplanken

Guardrails sind die Grenzen: Welche Aktionen darf der Agent ohne Rückfrage ausführen? Welche Eingaben/Ausgaben werden gefiltert? Wann braucht es ein menschliches „OK" (Human-in-the-Loop)? Guardrails sind teils Policy, teils Code – und eng mit Security verzahnt (Modul 9).

4.3 Warum das Harness 2026/2027 das Thema ist

Drei Entwicklungen treffen zusammen:

  • Modelle sind „gut genug". Der Reasoning-Sprung der letzten Generationen bedeutet: Der Engpass ist nicht mehr das Gehirn, sondern der Körper. Den größten Hebel hat heute, wer das Harness beherrscht.
  • Harness Engineering wird zur Disziplin. Es gibt mittlerweile etablierte Muster, Bibliotheken und „Awesome-Listen" rund um Harness-Engineering: Tools, Permissions, Memory, MCP, Observability, Orchestrierung. Was 2023 Bastelei war, ist 2026 Engineering mit Best Practices.
  • Standardisierung greift. MCP für Tool-Anbindung und OpenTelemetry für Telemetrie schaffen Interoperabilität. Das verschiebt den Wettbewerb von „welches Modell" zu „welches Harness".
Merksatz
2023–2025 war das Rennen um das beste Modell. 2026–2027 ist das Rennen um das beste Harness. Das Gehirn ist Commodity geworden – der Körper ist der Wettbewerbsvorteil.
